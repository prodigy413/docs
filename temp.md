# 運用手順書_アラーム有効/無効

## 前提

### 作業分類

+ 変更系作業

### 想定操作ユーザ・場所

+ ユーザ
  + 管理者権限にスイッチロール可能なIAMユーザー

+ 場所
  + NW環境
    + 開発NW / 運用ルーム
    + AWS管理コンソールへの接続：開発NW

### 事前に必要な作業

+ 管理者ロール(IAM_R_USR_<環境識別子>c_NAdmin_DevNW)にスイッチしておくこと。

### 実施タイミング

+ 必要な時に随時

### その他

+ 特になし

## アラーム無効

1. [検索]にてCloudWatchを検索し、[CloudWatch]サービスを選択する。
  ![画象サンプル](./material/アラーム有効無効/test.png)

2. 左ペインでアラーム[すべてのアラーム]をクリックし、アラームリストを表示する。
3. 右ペインで無効にするアラームにチェックを入れて[アクション] → [アラームアクション - 新規] → [無効化]をクリックする
4. 確認画面で[無効化]をクリックする
5. dfsgdsfg
6. dfsgdsfg
7. dsfgdfsg
    + 設定
      | 設定項目 | 設定する内容 |
      |:---|:---|
      | リソースタイプ | EC2 |

## 作業完了条件

+ アクションのステータスが「アクションが無効になっています」に変わること
+ アラートが発生してもアクションが発動しないこと
※アラーム設定の無効ではなくアクションを無効にするため、無効にした後もアラートは表示される。

## イベントルール変更

1. [検索]にてEventBridgeを検索し、[Amazon EventBridge]サービスを選択する。
  ![画象サンプル](./material/アラーム有効無効/test.png)

2. 左ペインでバース[ルール]をクリック > 右ペインでイベントバスをクリックして該当イベントバスを選択する
3. [ルール]項目で変更するルールをクリックし、該当ルールの画面を開く
4. [イベントパターン]タブ > [編集]をクリックし、編集画面を開く
5. [イベントパターン]項目で設定を更新し、[次へ]をクリックする。
6. [確認と更新にスキップ]をクリックする
7. 設定を確認し、[ルールを更新]をクリックする
    + 設定
      | 設定項目 | 設定する内容 |
      |:---|:---|
      | リソースタイプ | EC2 |

## 作業完了条件

+ イベントパターンが更新されていること
+ 該当アラートが発生すると通知が来ること
