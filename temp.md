# AWS Config / AWS Security Hub CSPM（シンプルバージョン）

## 目的

- 本設計書は、AWS Organizations 配下のマルチアカウント環境において、AWS ConfigとAWS Security Hub CSPMを利用した構成管理、コンプライアンス評価、セキュリティ統制の設計を定義する。
- AWS Config により、各アカウントのAWSリソース設定情報と変更履歴を記録し、組織全体の設定状態を確認できるようにする。
- AWS Security Hub CSPM により、組織全体へ共通のセキュリティ基準を適用し、セキュリティ検出結果を中央で管理する。

---

## 基本方針

- 本設計では、AWS Organizationsを前提に、管理・監査・ログ保管の責務を分離する。
- managementアカウントは、AWS Organizationsの管理および委任管理者の登録に利用する。
- auditアカウントは、AWS Configの集約確認およびSecurity Hub CSPMの中央管理に利用する。
- log-archiveアカウントは、AWS Configの設定履歴・スナップショットを保管するために利用する。

---

## 全体構成

- AWS Configは、対象アカウントで有効化する。
- 各アカウントのAWS Configは、AWSリソースの設定情報を記録し、設定履歴とスナップショットをlog-archiveアカウントのS3バケットへ配信する。
- auditアカウントでは、AWS Configの組織集約機能を利用し、AWS Organizations配下のアカウントおよびリージョンの設定情報を一元的に確認する。
- Security Hub CSPMは、auditアカウントを委任管理者として構成し、組織全体へ共通のセキュリティ標準を適用する。

```text
AWS Organizations
│
├── management account
│   └── Organizations 管理 / 委任管理者登録
│
├── audit account
│   ├── AWS Config 組織集約
│   └── Security Hub CSPM 中央管理
│
├── log-archive account
│   └── AWS Config ログ保管用 S3
│
└── member accounts
    ├── AWS Config 有効化
    └── Security Hub CSPM 評価対象
```

## AWS Config

### 利用目的

AWS Config は、AWSリソースの設定状態、変更履歴、コンプライアンス状況を管理するために利用する。

| 目的 | 内容 |
|---|---|
| 設定履歴の記録 | AWSリソースの設定変更履歴を保存する |
| コンプライアンス評価 | 管理ルールに基づいて設定準拠状況を評価する |
| 組織横断の可視化 | 複数アカウント・複数リージョンの設定状態を確認する |
| 監査証跡の保管 | 設定履歴を専用アカウントに保管する |

### 有効化対象

- AWS Configは、組織内の全アカウントで有効化する。
- 新規アカウントを追加する場合も、原則としてAWS Configを有効化する。

### 記録対象

AWS Configの記録対象は以下とする。

| 項目 | 設定値 |
|---|---|
| 記録対象 | サポートされるすべてのAWSリソース |
| グローバルリソース | 記録対象に含める |
| 記録リージョン | 利用リージョンを中心に有効化 |
| 配信先 | log-archiveアカウントのS3バケット |

IAMなどのグローバルリソースを記録対象に含める。ただし、複数リージョンで重複して評価される可能性があるため、運用時は評価結果の確認リージョンを明確にする。

### AWS Config ログ保管

AWS Configの設定履歴およびスナップショットは、log-archive アカウントのS3バケットへ保管する。

| 項目 | 設定値 |
|---|---|
| 保管先アカウント | log-archive |
| 保管先 | S3バケット |
| バケット名 | mbt-config-logs-01 |
| S3プレフィックス | <組織ID>/AWSLogs/<アカウントID>/Config/ |
| バージョニング | 有効 |
| 低コストストレージ移行 | 360日後にGlacierへ移行 |
| オブジェクト保管期間 | 1800日 |
| 非現行バージョン保管期間 | 360日 |
| 通信制御 | SSL通信のみ許可 |
| 書き込み制御 | AWS Config からの配信のみ許可 |
| 組織制限 | 同一AWS Organizationsからの配信のみ許可 |

### AWS Organizations 連携

AWS ConfigはAWS Organizationsと連携し、組織配下アカウントの設定情報を中央で管理できるようにする。

| 項目 | 設定値 |
|---|---|
| 組織連携 | 有効 |
| 委任管理者 | audit アカウント |
| 集約対象 | AWS Organizations 配下の対象アカウント |
| 集約リージョン | 全リージョン |

### 組織集約

AWS Configの集約はauditアカウントで実施する。

| 項目 | 設定値 |
|---|---|
| 集約先アカウント | audit |
| 集約名 | organization-config-aggregator |
| 集約対象 | AWS Organizations 配下の対象アカウント |
| 対象リージョン | 全リージョン |
| 用途 | リソース設定状態、変更履歴、準拠状況の一元確認 |

### コンプライアンス評価項目

初期導入時のAWS Config評価項目は以下とする。

| 評価項目 | 目的 | 適用範囲 |
|---|---|---|
| CloudTrail 有効化確認 | 監査ログ取得が有効であることを確認 | 組織全体 |
| Security Hub CSPM 有効化確認 | セキュリティ評価基盤が有効であることを確認 | 組織全体 |
| rootアカウントMFA確認 | rootアカウント保護を確認 | 対象アカウント |
| IAMユーザーMFA確認 | IAMユーザーの認証強化を確認 | 対象アカウント |
| IAMパスワードポリシー確認 | パスワード強度の基準を確認 | 対象アカウント |

IAM Identity Centerを認証基盤として利用するが、IAM管理者ユーザーを利用するため、IAMに関する評価項目についても継続して適用する。

### IAMパスワードポリシー基準

IAMユーザーを利用する場合のパスワードポリシーは以下を基準とする。

| 項目 | 設定値 |
|---|---|
| 英大文字 | 必須 |
| 英小文字 | 必須 |
| 数字 | 必須 |
| 記号 | 必須 |
| 最小文字数 | 14文字以上 |
| パスワード再利用防止 | 24世代 |
| パスワード有効期限 | 90日 |

## AWS Security Hub CSPM

### 利用目的

- AWS Security Hub CSPMは、AWS環境のセキュリティベストプラクティス評価と検出結果の一元管理を目的として利用する。
- AWS Organizationsと連携し、組織全体に共通のセキュリティ標準を適用する。

### 管理方式

Security Hub CSPMは、auditアカウントを委任管理者として中央管理する。

| 項目 | 設定値 |
|---|---|
| 委任管理者 | auditアカウント |
| 管理方式 | 中央設定 |
| 自動有効化 | 無効 |
| 標準の自動有効化 | 無効 |
| 適用方式 | 中央設定ポリシーにより適用 |

各アカウントで個別にSecurity Hub CSPM設定を管理するのではなく、auditアカウントで定義した共通設定を組織全体へ適用する。

### ホームリージョン

Security Hub CSPMのホームリージョンは以下とする。

| 項目 | 設定値 |
|---|---|
| ホームリージョン | ap-northeast-1 |
| 主な利用目的 | Security Hub CSPM の中央管理 |
| 運用主体 | auditアカウント |

### 検出結果のリージョン集約

現時点では、Security Hub CSPMの検出結果はホームリージョンを中心に運用する。

| 項目 | 設定値 |
|---|---|
| リージョン集約 | 他リージョン連携なし |
| 中心リージョン | ap-northeast-1 |
| 将来対応 | 複数リージョン利用時に集約対象を追加検討 |

東京リージョン以外も本格利用する場合は、検出結果をどのリージョンへ集約するかを再設計する。

### 適用するセキュリティ標準

初期導入時は、以下のセキュリティ標準を適用する。

| 項目 | 設定値 |
|---|---|
| セキュリティ標準 | CIS AWS Foundations Benchmark |
| バージョン | v5.0.0 |
| 適用範囲 | AWS Organizations Root 配下 |
| 無効化コントロール | 初期状態ではなし |

初期導入時は、セキュリティ設定要件書で評価対象とするセキュリティ標準のみを有効化する。今後、追加の評価要件が発生した場合は、必要に応じて適用するセキュリティ標準を追加する。

### 適用範囲

Security Hub CSPMの共通設定は、AWS Organizations Rootに適用する。

| 項目 | 設定値 |
|---|---|
| 適用単位 | AWS Organizations Root |
| 対象 | 配下の対象アカウント |
| 例外 | 必要に応じてOUまたはアカウント単位で管理 |

## 運用・注意事項など

### 新規アカウント追加時の確認

AWS Organizationsに新規アカウントを追加した場合、以下を確認する。

| 確認項目 | 内容 |
|---|---|
| AWS Config | 対象アカウントで有効化されていること |
| Configログ配信 | log-archiveアカウントへ配信されていること |
| 組織集約 | auditアカウントで確認できること |
| Security Hub CSPM | 共通ベースラインが適用されていること |
| CIS評価 | 評価が開始されていること |

### グローバルリソースの記録

- IAMなどのグローバルリソースは、複数リージョンで重複して記録・評価される可能性がある。
- そのため、運用時はグローバルリソースの評価結果をどのリージョンで確認するかを明確にする。

### Security Hub CSPM のリージョン拡張

- 現時点では東京リージョン中心の運用とする。
- 今後、他リージョンを本格利用する場合は、Security Hub CSPMのリージョン集約設定を見直す。

### ログ保管保護の強化

- AWS Config ログ保管用S3バケットは、監査証跡の保管先である。
- 本番運用では、S3バケットの削除防止、バケットポリシー変更制限、ライフサイクル変更制限などをSCPまたは権限設計で保護することを検討する。
